Курсовой проект по дисциплине «Разработка веб-приложений». 
Представляет собой полноценный маркетплейс электроники с изолированными ролевыми моделями: **Покупатель**, **Продавец** и **Администратор**.

 **[Перейти на сайт](https://frontendcp.vercel.app/)** 

---

## Структура репозитория

Репозиторий разделен на две логические части: программный код и отчетная документация.

* 📂 **`code/frontend/`** — Клиентское SPA-приложение (React, Vite, Tailwind CSS).
* 📂 **`code/backend/`** — Серверный API (FastAPI, Python, SQLAlchemy, JWT).
* 📂 **`docs/`** — Исходный код LaTeX и скомпилированная пояснительная записка в формате PDF.

---

## Ключевые архитектурные решения
* **Динамическая фильтрация:** Характеристики товаров хранятся в PostgreSQL в типе данных `JSONB`. На клиенте реализован алгоритм фильтрации, который автоматически генерирует селекторы.
* **Транзакционная корзина:** Оформление заказа со строгим списанием складских остатков и безопасным откатом транзакции в случае ошибок.
* **Ролевая модель (RBAC):** Авторизация по стандарту OAuth2. JWT-токены и класс-зависимость `RoleChecker` для защиты эндпоинтов на стороне сервера.
* **B2B Дашборды:** Изолированные панели для продавцов с графиками, экспортом отчетов в CSV и конструктором товаров, а для администраторов управление справочником категорий и возможность смены ролей.

### 1. Запуск сервера
Необходим установленный Python версии 3.10+.
```bash
cd code/backend
python -m venv venv
source venv/bin/activate  # Для Windows используйте: venv\Scripts\activate
pip install -r requirements.txt
```
*Создайте файл `.env` в папке `backend` с настройками подключения к вашей СУБД PostgreSQL (переменные `DB_HOST`, `DB_USER`, `SECRET_KEY`...).*
```bash
# Запуск сервера
uvicorn app.main:app --reload
```
Документация API Swagger будет доступна по адресу `http://localhost:8000/docs`. Там же будет предоставлена возможность протестировать работу эндпоинтов.

### 2. Запуск клиента
Необходим установленный Node.js.
```bash
cd code/frontend
npm install
npm run dev
```
Приложение откроется в браузере по адресу `http://localhost:5173/`.
